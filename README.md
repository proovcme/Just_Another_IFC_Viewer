# BIM AI Viewer (v8.0.1)

**Статус:** Production (Stable)  
**Стек:** Vanilla JS, Three.js, web-ifc, web-ifc-three, PHP (backend-lite)

Легкий и быстрый веб-вьюер для BIM-моделей (IFC), написанный на Vanilla JS с использованием Three.js и ядра web-ifc. Поддерживает загрузку сводных моделей (Federated BIM) и турбо-индексацию инженерных систем.

---

## 📖 Особенности

* **Производительность:** Аппаратное ускорение через WebGL/Three.js и парсинг IFC на базе WASM (`web-ifc`).
* **Мультимодельность:** Загрузка нескольких `.ifc` файлов одновременно с построением единого сводного дерева проекта.
* **Турбо-индексация:** Мгновенный сбор атрибутов "Имя системы" (System Name) и "Семейство/Типоразмер" (Family and Type) из графа свойств (Psets) без зависания UI. Поддержка как IFC2x3, так и IFC4.
* **Инженерная изоляция:** Группировка элементов по системам (Вентиляция, Трубопроводы, Оборудование) с визуальным выделением в 1 клик.
* **Инструменты работы с геометрией:** 
  * Интерактивное пространственное дерево (Spatial Structure).
  * Рулетка (измерения ΔX, ΔY, ΔZ и абсолютного расстояния).
  * Инструменты сечения (X/Y плоскости).
  * Режимы отображения: X-Ray (прозрачность), Ортографическая камера.
* **Режим прораба:** WASD-управление камерой от первого лица (Minecraft-style).
* **Engineering Noir UI:** Строгий монохромный интерфейс без визуального шума.

## ⚙️ Алгоритмы

### 1. Турбо-индексация свойств
**Проблема:** Вызов `getPropertySets()` для каждого элемента убивает браузер на моделях >4000 элементов.  
**Решение:**
1. **Массовая выгрузка:** Единый запрос `getAllProperties(modelID)` или обход графа `IfcRelDefinesByProperties`.
2. **Парсинг в потоке:** Синхронный разбор JS-массива свойств с поиском ключей:
   - `"Имя системы"` (Revit mapping)
   - `"System Name"` (IFC Standard)
   - `"SystemType"` (IFC Building System)
3. **Очистка имен:** Удаление мусора (GUID, ID семейств Revit, координаты уровней) через регекспы.
4. **Группировка:** Распределение по категориям (Вентиляция, Трубопроводы, Оборудование, Уровни).
5. **DOM-привязка:** Запись результата в атрибут `data-system` узлов дерева.

```javascript
// Пример очистки имени системы
cleanSystemName(rawName) {
    if (!rawName) return null;
    // Удаляем Revit Element IDs (:123456), координаты (_+0.000_), технические префиксы
    let cleaned = rawName.replace(/:\d+$/, '').replace(/_?\+[0-9.]+_?/, '');
    // Фильтруем уровни и мусор
    if (/^УРОВЕНЬ|FLOOR|LEVEL|^00-/.test(cleaned)) return null;
    return cleaned.trim().toUpperCase();
}
```

### 2. Визуальная изоляция систем
**Алгоритм:**
1. Пользователь кликает на тег системы (например, "Вентиляция").
2. Скрипт сканирует дерево, собирая `expressID` всех узлов с `data-system="ВЕНТИЛЯЦИЯ"`.
3. Группировка по `modelID` (для Federated BIM).
4. Для каждой модели создается `Subset` через `manager.createSubset({ modelID, ids, customID })`.
5. Остальная геометрия скрывается или делается полупрозрачной (X-Ray режим).
6. При повторном клике: `manager.removeSubset()` и сброс видимости.

### 3. Обработка двойного клика (Raycasting)
1. Перехват события `dblclick` на `<canvas>`.
2. Нормализация координат мыши в NDC (-1 до +1).
3. Пуск луча `Raycaster` через видимые меши моделей.
4. Получение `expressID` из хита: `modelID = hit.object.modelID`, `id = getExpressId(geometry, faceIndex)`.
5. Создание временного `Subset` для подсветки.
6. Открытие панели свойств с данными элемента.

## 🐛 Известные проблемы и решения

| Проблема | Статус | Решение / Комментарий |
|----------|--------|-----------------------|
| **Отсутствие `IfcSystem` в некоторых IFC** | ⚠️ Частично решено | Revit часто не экспортирует класс `IfcSystem`, заменяя его на Pset "Имя системы". Реализован fallback на парсинг Psets. Требуется доработка маппинга RVT → IFC. |
| **Мусор в именах систем** | ✅ Решено | Регексп-фильтры удаляют GUID, координаты уровней и суффиксы семейств (`:12345`). Группировка по категориям. |
| **`manager.getAllLines` undefined** | ✅ Исправлено | Метод удален из кода. Сбор ID теперь идет напрямую из DOM-дерева. |
| **Блокировка UI при индексации** | ✅ Решено | Использование `async/await` с `await new Promise(r => setTimeout(r, 0))` для разгрузки потока. |
| **Ошибки Raycaster при клике** | ✅ Исправлено | Проверка `hit.object.layers.test(0)` и фильтрация не-Mesh объектов. |

## 🛠 Установка и запуск

### Требования
- PHP 7.4+ (только для сканирования папки `models/`)
- Современный браузер (Chrome 90+, Firefox 88+)

### Локальный запуск
```bash
# Клонируйте репозиторий
cd /workspace

# Запустите встроенный PHP сервер
php -S localhost:8000

# Откройте в браузере
http://localhost:8000
```

### Добавление моделей
Поместите `.ifc` файлы в папку `/workspace/models`. Они автоматически появятся в списке выбора при загрузке страницы.
